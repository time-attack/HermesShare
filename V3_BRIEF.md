## Transport (already working, don't re-litigate)
Two ways to send a card into a real iMessage conversation, both proven working today:
- **Linq API** (`POST https://api.linqapp.com/api/partner/v3/chats`, `imessage_app` message part) — requires `url` to be `data:application/json;base64,<compact-json-b64>` (Linq rejects custom URL schemes). Supports `layout.image_url` for a real preview thumbnail — BUT this needs (a) an established chat (recipient has texted the Linq number at least once) and (b) currently returns 403 "raw iMessage app cards not available" unless Linq support flips an account flag (email already sent to their rep, Josh, asking for this — check `~/.hermes/skills/productivity/hermesshare-cards/SKILL.md` troubleshooting section for the exact error/ask).
- **Photon** (`spectrum-ts`, cloud/dedicated mode) via `customizedMiniApp()` — requires `url` to be a real `https://` URL with a `?p=<base64url-json>` query param (Photon silently fails to set `MSMessage.url` on-device for custom schemes like `hermesshare://` — confirmed via on-device debug logging, NOT documented anywhere by Photon). Photon's `layout` object has NO `image_url` field at all (only `caption`/`subcaption`/`trailingCaption`) — so Photon-sent cards can never show a thumbnail, full stop, regardless of anything else. `~/.hermes/hermes-agent/plugins/platforms/photon/sidecar/send_card_photon_test.mjs` is a working reference sender (reads `PHOTON_PROJECT_ID`/`PHOTON_PROJECT_SECRET` env vars from `~/.hermes/.env`).

Both `MessagesViewController.layout(from:)` decode paths (data: URL, and any URL with a `?p=` query item) are implemented and proven working via on-device debug logs.
